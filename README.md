# Log Viewer API

Простой API для чтения и отдачи логов. Сервис только читает существующие файлы журналов и не записывает их.

## Что делает

- Отдает содержимое логов в виде текста или JSON
- Не предоставляет возможности записи, удаления или изменения логов
- Имеется страница для аутентификации пользователей

## Основные маршруты

- `GET /api/logs` - список доступных логов
- `GET /api/logs/{filename}` - получить содержимое указанного файла лога
- `GET /login` - страница аутентификации
- `POST /login` - отправка данных для входа

## Поведение

- Все запросы к `/api/logs` работают только на чтение
- Логи считываются из заранее настроенной директории
- API не создает, не редактирует и не удаляет файлы логов

## Страница аутентификации

Страница `GET /login` должна отображать форму для ввода логина и пароля.
После успешной авторизации пользователь получает доступ к списку логов и содержимому файлов.

## Настройка

1. Укажите корневую директорию логов в конфигурации приложения
2. Настройте метод аутентификации (сессии, JWT или другой)
3. Запустите сервис и откройте страницу `/login`

## Примечание

API предназначено только для просмотра логов. Все операции записи и управления файлами должны выполняться вне этого сервиса.
